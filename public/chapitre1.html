Git

Introduction à Git

Git est un outil essentiel, que vous travailliez seul, en équipe, ou même sur un projet en open source.Il permet de suivre les modifications de votre code et d'organiser vos projets de développement et de pouvoir les rétablir à tout moment.Les branches correspondent à une version de votre projet qui évolue de manière indépendante. Cela permet de pouvoir expérimenter des solutions sans impacter le code existant.La branche "principale" d'un projet est souvent appelée "master" ou "main". Cette branche doit être la plus saine possible.

Création d'un dépot git

  • cd project
  • git init

Ajout des fichiers

On peut ajouter les fichiers individuellement (1) ou tous les fichiers du projet (2).

  1. git add 'nomdufichier'
  2. git add.

Faire un commit

Un commit est comme une sorte de point d'ancrage qui permet au dev de sauvegarder une version du code à un instant T avec une description de son travail. Un commit est identifié de manière unique via un hash qui lui est attribué.

La commande est la suivante : git commit -a -m "message du commit"

  • -a pour add (on ajoute au suivi les nouveaux fichiers créés s'il y en a)
  • -m pour message (on décrit les modifications apportées au code)

Un commit doit toujours être le plus explicite possible tout en étant concit dans sa description.

Attention, vous devez au préalable configurer votre git local afin qu'il puisse vous reconnaitre avant de pouvoir réaliser un commit. les étapes suivantes doivent être réaliser une seule fois.

  • git config --global user.name "username"
  • git config --global user.email "votreemail@email.fr"
